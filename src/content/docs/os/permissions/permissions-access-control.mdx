Every file and process on an OS has an owner and a set of permissions. The OS enforces these at the kernel level — even root can be restricted by certain mechanisms.

## Unix / Linux Permissions

### The Permission Model

Every file has:
- An **owner** (a user)
- A **group** (a user group)
- A **permission set** for owner, group, and others

```
-rwxr-xr--  1  alice  developers  4096  May 20 10:00  script.sh
│└──┴──┴──      │       │
│ owner group others   owner  group
│
└ file type: - = file, d = directory, l = symlink
```

### Permission Bits

| Symbol | Octal | Meaning on file | Meaning on directory |
|---|---|---|---|
| `r` | 4 | Read file contents | List directory contents |
| `w` | 2 | Write / modify file | Create, delete, rename files inside |
| `x` | 1 | Execute file | Enter (cd into) directory |
| `-` | 0 | No permission | No permission |

**Reading permissions:** `rwxr-xr--`
- Owner: `rwx` = read + write + execute
- Group: `r-x` = read + execute (no write)
- Others: `r--` = read only

### chmod — Change Permissions

```bash
# Symbolic mode
chmod u+x script.sh        # add execute for owner
chmod g-w file.txt         # remove write from group
chmod o=r file.txt         # set others to read only
chmod a+r file.txt         # all (owner+group+others) add read

# Octal mode
chmod 755 script.sh        # rwxr-xr-x (common for executables)
chmod 644 file.txt         # rw-r--r-- (common for files)
chmod 600 private.key      # rw------- (private files)
chmod 700 ~/.ssh           # rwx------ (private directory)
```

### chown — Change Owner

```bash
chown alice file.txt            # change owner
chown alice:developers file.txt # change owner and group
chown -R alice:dev /opt/app     # recursive
```

## Special Permission Bits

| Bit | Name | Effect on file | Effect on directory |
|---|---|---|---|
| `s` on user | SetUID (SUID) | Runs as file owner, not current user | — |
| `s` on group | SetGID (SGID) | Runs as file group | New files inherit directory's group |
| `t` on others | Sticky bit | — | Only owner can delete their files |

```bash
# SUID example: /usr/bin/passwd runs as root even if called by a normal user
ls -l /usr/bin/passwd
# -rwsr-xr-x root root /usr/bin/passwd

# Sticky bit: /tmp — anyone can write, but only delete their own files
ls -ld /tmp
# drwxrwxrwt root root /tmp

chmod +t /shared/dir          # add sticky bit
chmod u+s /usr/local/bin/app  # add SUID
```

## Access Control Lists (ACLs)

Standard Unix permissions only allow one owner and one group. **ACLs** let you set permissions for any number of users or groups on a single file.

```bash
# View ACL
getfacl /opt/app/config.txt

# Add read access for user bob
setfacl -m u:bob:r /opt/app/config.txt

# Add write access for group auditors
setfacl -m g:auditors:rw /opt/app/logs/

# Default ACL (inherited by new files in directory)
setfacl -d -m g:developers:rw /opt/app/

# Remove an entry
setfacl -x u:bob /opt/app/config.txt

# Remove all ACLs
setfacl -b /opt/app/config.txt
```

Files with ACLs show a `+` in `ls -l`: `-rw-r--r--+`

## Windows NTFS Permissions

NTFS uses **Access Control Lists (ACLs)** for every file and directory. More granular than Unix rwx.

### Basic Permissions

| Permission | Allows |
|---|---|
| Full Control | Read, write, modify, delete, change permissions |
| Modify | Read, write, delete |
| Read & Execute | Read and run executables |
| Read | View file contents and attributes |
| Write | Create and modify files |

### icacls (Command Line)

```powershell
# View permissions
icacls C:\Users\alice\Documents

# Grant write permission to bob
icacls C:\data\reports /grant bob:W

# Revoke all permissions for bob
icacls C:\data\reports /remove bob

# Reset to inherited permissions
icacls C:\data /reset /T

# Set inheritance
icacls C:\data /inheritance:d    # disable inheritance
icacls C:\data /inheritance:e    # enable inheritance
```

### Permission Inheritance

By default, child files and folders **inherit** permissions from the parent. You can break inheritance to set custom permissions, or force-propagate permissions down a tree.

## Linux sudo

`sudo` lets a standard user run specific commands as root, logged and controlled via `/etc/sudoers`.

```bash
sudo apt install nginx          # run as root
sudo -u postgres psql           # run as a specific user
sudo -i                         # open root shell

# /etc/sudoers (edit with visudo):
alice   ALL=(ALL) ALL           # alice can run any command as any user
bob     ALL=(ALL) NOPASSWD: /bin/systemctl restart nginx  # no password for specific cmd
%devops ALL=(ALL) ALL           # group devops has full sudo
```

## Summary

| Feature | Linux | Windows |
|---|---|---|
| Basic model | rwx × (owner, group, others) | ACL with named users/groups |
| Extended ACLs | POSIX ACLs (`setfacl`) | NTFS ACLs (always available) |
| Admin escalation | `sudo` | UAC + Run as Administrator |
| Audit logging | `/var/log/auth.log` | Security Event Log |
