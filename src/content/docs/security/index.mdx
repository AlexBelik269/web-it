## Defense in Depth

A modern system should assume that **any single control can fail**.

<Steps>

1. **Protect the perimeter**
   - Firewalls
   - DDoS protection
   - IDS/IPS
   - Network segmentation

2. **Harden the application**
   - Input validation
   - Output encoding
   - CSP headers
   - WAF protections

3. **Secure identity & access**
   - MFA
   - Least privilege
   - Session security
   - Role-based authorization

4. **Protect sensitive data**
   - Encryption at rest
   - TLS in transit
   - Parameterized queries
   - Secrets management

5. **Detect attacks early**
   - Centralized logging
   - Monitoring & alerting
   - SIEM pipelines
   - Endpoint detection

6. **Respond & recover**
   - Incident playbooks
   - Backups
   - Forensics
   - Disaster recovery

</Steps>

## Core Security Principles

| Principle | Meaning |
|---|---|
| Least Privilege | Users and services should only have the minimum access required |
| Defense in Depth | Multiple independent layers of security |
| Zero Trust | Never automatically trust internal or external systems |
| Fail Securely | Systems should fail safely without exposing sensitive data |
| Secure Defaults | Security should be enabled by default |
| Visibility & Detection | You cannot defend what you cannot observe |

<Aside type="tip" title="Practical mindset">
Security is not just about preventing attacks — it’s about reducing risk, limiting blast radius, detecting compromise quickly, and recovering safely.
</Aside>
